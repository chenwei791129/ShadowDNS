## 6. Parser 支援 include 檔內的 options{}（apply/code-review 時發現的必要前提）

**背景**：`make smoke`（真實 CLI 路徑）實測失敗 `geoip-directory is not set`。根因為 `internal/config/zones.go` 原以 `if cfg.Path == path` **只套用 root named.conf 的 options{}**，靜默丟棄被 include 檔案內的 options{}。Debian split 把 options 搬進 `named.conf.options`，導致 directory/geoip-directory/listen-on/rate-limit/allow-transfer 全被丟棄；整合測試因直接注入 GeoIP handle、且 zone 檔與宣告檔同層而遮蔽此問題。出貨範本 `named.conf.options.example` 照抄會產生壞設定。經使用者裁示「擴充 parser 支援 include options」。

- [x] 6.2 `internal/config/zones_test.go`：新增 `TestLoadNamedConf_OptionsFromIncludedFile`（驗證被 include 的 options{} 的 directory/geoip-directory/listen-on 都生效）與 `TestLoadNamedConf_MultipleOptionsBlocksLastWins`（多區塊 last-wins＋警告）
