## 1. 共用原語 `internal/httpserver`（Decision 1：共用原語放在新 package `internal/httpserver`，只負責單台 server）

- [x] 1.1 建立 `internal/httpserver` package，導出單一 graceful-shutdown deadline 常數（值維持 5s）與一個建構 hardened `*http.Server` 的 helper（套用 read 10s / idle 120s / header 5s；WriteTimeout 預設 10s 但**允許 caller 省略／覆寫**，供 metrics 的 pprof 串流例外）。交付「Single source of truth for graceful-shutdown deadline」與「Hardened connection timeouts on all background HTTP servers」的共用來源（Decision 3：graceful-shutdown deadline 與 hardened 逾時收斂為單一來源）。驗證：`go build ./internal/httpserver` 通過，且常數/helper 為 exported。
- [x] 1.2 實作 serve 函式：在 goroutine 啟動「caller 注入的 serve-start closure（`func() error`）」、`select` on `ctx.Done()` / `errCh`、停機時以 **fresh `context.WithTimeout(context.Background(), 共用 deadline)`**（不可用已取消的 parent ctx）呼叫 `Shutdown`、drain `errCh`、過濾並回傳第一個非 `http.ErrServerClosed` 錯誤、正常停機回傳 `nil`。交付「Serve helper reports the first real serve error」。驗證：closure 形式可同時服務 `srv.Serve(ln)`（API）、`srv.ListenAndServe()`（metrics）與 `srv.ListenAndServeTLS("", "")`（DoH HTTPS，TLS + `GetCertificate`）三種 caller。
- [x] 1.3 導出一個小型 per-server drain helper（`Shutdown(共用 deadline) + Warn-on-failure`），供 DoH 並行 drain goroutine 取代自有的 `shutdownServer`，避免 drain 行為在兩處漂移（Decision 2：DoH 的多 listener 與憑證續期協調，疊在共用原語之上）。驗證：`go build ./internal/httpserver` 通過。
- [x] 1.4 為 `internal/httpserver` 撰寫單元測試，涵蓋 (a) ctx 取消觸發 graceful shutdown 且 in-flight request 完成（驗證未用已取消 ctx 導致 no-op）、(b) serve 失敗回傳非 `ErrServerClosed` 錯誤、(c) `ErrServerClosed` 被吸收為 `nil`，覆蓋「Serve helper reports the first real serve error」。驗證：`go test ./internal/httpserver -race` 全綠。
